# 빈자리 MVP 스프린트 계획

> Version 0.2
>
> Last Updated: 2026-07-13
>
> 목표: 3개월 내 서울 강남·마포·성수 지역 MVP 출시

---

# 전체 일정

```
Week 1~2   : 환경 세팅 + 인증 + 공간 등록 (진행 완료)
Week 3~4   : 공간 검색 + 상세 화면 (진행 완료)
Week 5~6   : 예약 + 결제 (진행 완료)
Week 7~8   : 알림 + 리뷰 + 정산 (진행 중)
Week 9~10  : 관리자 + QA
Week 11~12 : 버그 수정 + 출시 준비
```

---

# Sprint 1 — 기반 세팅 (Week 1~2)

> 목표: 개발 환경 구축, 회원 인증, 공간 등록 완료

## 백엔드

- [x] 프로젝트 초기 세팅 (NestJS + PostgreSQL)
- [x] DB 스키마 설계 (User, Space, Booking, Review)
- [x] 카카오 OAuth 연동
- [ ] 구글 OAuth 연동
- [ ] 사업자등록번호 인증 API (국세청)
- [x] JWT 인증 미들웨어
- [x] 공간 등록 API (CRUD)
- [x] 파일 업로드 API — 로컬 스토리지 기반 구현 완료

## 프론트엔드 (웹)

- [x] Next.js 프로젝트 세팅 (TypeScript, App Router)
- [x] 모바일 최적화 웹 반응형 레이아웃 구성 (모바일 뷰포트 고정)
- [x] 서비스 소개 및 랜딩 페이지
- [x] 로그인 / 회원가입 (카카오 소셜 로그인 연동)
- [ ] 사업자 인증 정보 입력 폼 (Host)
- [x] 공간 등록 Step 1 — 사진 업로드 컴포넌트
- [x] 공간 등록 Step 2 — 지도 위치 선택 (카카오맵 웹 SDK 연동)
- [x] 공간 등록 Step 3 — 가격 및 이용 가용 캘린더 입력
- [x] 공간 등록 Step 4 — 시설 종류 및 이용 규칙 설정
- [x] 공간 등록 완료 페이지

## 완료 기준

- Host가 공간을 등록할 수 있다
- 카카오 소셜 로그인이 된다
- 등록된 공간이 DB에 저장된다

- [x] 공간 상세 API
- [ ] 공간 승인 프로세스 (어드민 → 게시)
- [ ] 운영팀 어드민 페이지 (공간 승인/거절)

## 프론트엔드 (웹)

- [x] 홈 화면 — 지도 중심 UI (카카오맵 웹 SDK 연동 및 마커 표시)
- [ ] 상세 필터 모달 (날짜, 가격, 시설 조건)
- [x] 공간 상세 페이지 — 이미지 캐러셀 (고화질 슬라이드쇼 완료)
- [x] 공간 상세 페이지 — 시설 안내 및 역지오코딩된 주소 렌더링
- [x] 공간 상세 화면 — 정보 (시설, 규칙, 위치)
- [x] 공간 상세 화면 — 리뷰 섹션 (실시간 0.5단위 SVG 마스크 별점 피드)
- [x] 공간 상세 화면 — 가격 계산기 + 예약 버튼

- 날짜·가격 필터로 공간을 좁힐 수 있다
- 공간 상세 정보를 볼 수 있다

# Sprint 3 — 예약 + 결제 (Week 5~6)

> 목표: 예약 요청부터 결제까지 핵심 플로우 완성

## 백엔드

- [x] 예약 요청 API (잔여석 가용성 검증 및 동시성 방어 포함)
- [ ] 예약 수락/거절 API (Host)
- [x] 가상 결제 연동 (가상 카드 결제 승인 API 완료)
- [ ] 결제 웹훅 처리
- [x] 예약 취소 + 환불 처리 (일일 5회 한도 및 과거 날짜 방어 안티 어뷰징 가드 장착 완료)
- [x] 예약 내역 조회 API (리뷰 조인 포함)
- [ ] 입실 안내 자동 문자 발송 (알리고 or 네이버 클라우드)

## 프론트엔드 (웹)

- [x] 예약 입력 화면 — 날짜 선택 캘린더 및 증감 좌석 선택 수량 조작기
- [ ] 예약 입력 화면 — 메시지 입력
- [x] 결제 화면 — 금액 확인, 결제 수단 선택 (가상 결제 팝업 모달 완료)
- [x] 예약 완료 화면 (내 예약판 리다이렉트 완료)
- [ ] Host 홈 — 예약 요청 목록
- [ ] Host — 예약 수락/거절 화면
- [x] Guest 예약 내역 화면 (내역판 /bookings/my 완료)
- [x] 예약 취소 화면 (내역판 내 실시간 취소 및 뱃지 업데이트 완료)

## 완료 기준

- Guest가 공간을 예약 요청할 수 있다
- Host가 수락/거절할 수 있다 (현재 가상 결제 시 자동 확정 구조)
- 결제가 완료된다
- 취소 및 환불이 처리된다

# Sprint 4 — 알림 + 리뷰 + 정산 (Week 7~8)

> 목표: 이용 완료 후 리뷰, Host 정산, 알림 시스템

## 백엔드

- [ ] Firebase FCM 푸시 알림 연동
- [ ] 알림 발송 (예약 요청, 확정, 취소, 리뷰 요청)
- [x] 리뷰 작성 API (유효 확인 및 0.5점 단위 실수형 저장 지원)
- [x] 평균 평점 집계 (소수점 첫째자리 실시간 집계 stats API 완료)
- [ ] 정산 계좌 등록 API
- [ ] 월 정산 처리 (수동 → 자동화)
- [ ] 정산 내역 API

## 프론트엔드 (웹)

- [x] 리뷰 작성 화면 (별점 + 텍스트) (0.5 단위 SVG clipPath 마스크 터치/클릭 조작 완료, 슬라이더 바 제거)
- [ ] Host — 받은 리뷰 목록
- [ ] 알림 리스트 화면
- [ ] Host — 정산 계좌 등록 화면
- [ ] Host — 정산 내역 화면
- [x] 마이페이지 (내 예약 리스트 통합)

## 완료 기준

- 이용 완료 후 리뷰 작성이 정상 작동한다
- 브라우저 알림 또는 문자(SMS)로 즉각 알림이 전송된다
- Host가 정산 정보를 등록하고 매출 상태를 확인 가능하다

# Sprint 5 — QA + 버그 수정 (Week 9~10)

## 테스트 시나리오

- [ ] Guest 회원가입 → 검색 → 예약 → 결제 → 리뷰 작성 전체 플로우
- [x] 예약 취소 시 환불 처리 확인 (완료)
- [x] 지도 기반 검색 성능 및 반응형 뷰 테스트 (완료)
- [ ] 결제 실패 케이스 처리 확인
- [ ] 모바일 웹 브라우저 알림 및 SMS 문자 발송 연동 테스트

## 보안 체크
