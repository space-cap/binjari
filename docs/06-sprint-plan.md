# 빈자리 MVP 스프린트 계획

> Version 0.1
>
> Last Updated: 2026-07-12
>
> 목표: 3개월 내 서울 강남·마포·성수 지역 MVP 출시

---

# 전체 일정

```
Week 1~2   : 환경 세팅 + 인증 + 공간 등록
Week 3~4   : 공간 검색 + 상세 화면
Week 5~6   : 예약 + 결제
Week 7~8   : 알림 + 리뷰 + 정산
Week 9~10  : 관리자 + QA
Week 11~12 : 버그 수정 + 출시 준비
```

---

# Sprint 1 — 기반 세팅 (Week 1~2)

> 목표: 개발 환경 구축, 회원 인증, 공간 등록 완료

## 백엔드

- [ ] 프로젝트 초기 세팅 (NestJS + PostgreSQL)
- [ ] DB 스키마 설계 (User, Space, Booking, Review)
- [ ] 카카오 OAuth 연동
- [ ] 구글 OAuth 연동
- [ ] 사업자등록번호 인증 API (국세청)
- [ ] JWT 인증 미들웨어
- [ ] 공간 등록 API (CRUD)
- [ ] 파일 업로드 API — 개발: 로컬 스토리지 / 운영: OCI Object Storage (환경변수로 전환)

## 프론트엔드 (앱)

- [ ] React Native 프로젝트 세팅
- [ ] 네비게이션 구조 세팅
- [ ] 스플래시 / 온보딩 화면
- [ ] 로그인 화면 (카카오 소셜)
- [ ] 회원가입 (Host/Guest 선택)
- [ ] 사업자 인증 화면 (Host)
- [ ] 공간 등록 Step 1 — 사진 업로드
- [ ] 공간 등록 Step 2 — 위치 설정 (카카오맵)
- [ ] 공간 등록 Step 3 — 가격·가용일정 설정
- [ ] 공간 등록 Step 4 — 시설·규칙 설정
- [ ] 등록 완료 화면

## 완료 기준

- Host가 공간을 등록할 수 있다
- 카카오 소셜 로그인이 된다
- 등록된 공간이 DB에 저장된다

---

# Sprint 2 — 검색 + 상세 (Week 3~4)

> 목표: Guest가 공간을 검색하고 상세 정보를 볼 수 있다

## 백엔드

- [ ] 공간 검색 API (위치 기반, 날짜 필터, 가격 필터)
- [ ] 지도 마커 API (위·경도 기반 공간 목록)
- [ ] 공간 상세 API
- [ ] 공간 승인 프로세스 (어드민 → 게시)
- [ ] 운영팀 어드민 페이지 (공간 승인/거절)

## 프론트엔드 (앱)

- [ ] 홈 화면 — 지도 탭 (카카오맵 마커)
- [ ] 검색 바 + 날짜 필터
- [ ] 가격·시설 필터 UI
- [ ] 공간 카드 컴포넌트 (썸네일, 가격, 평점)
- [ ] 검색 결과 리스트 뷰
- [ ] 공간 상세 화면 — 사진 갤러리
- [ ] 공간 상세 화면 — 정보 (시설, 규칙, 위치)
- [ ] 공간 상세 화면 — 리뷰 섹션
- [ ] 공간 상세 화면 — 가격 계산기 + 예약 버튼

## 완료 기준

- Guest가 지도에서 주변 공간을 볼 수 있다
- 날짜·가격 필터로 공간을 좁힐 수 있다
- 공간 상세 정보를 볼 수 있다

---

# Sprint 3 — 예약 + 결제 (Week 5~6)

> 목표: 예약 요청부터 결제까지 핵심 플로우 완성

## 백엔드

- [ ] 예약 요청 API
- [ ] 예약 수락/거절 API (Host)
- [ ] 토스페이먼츠 결제 연동
- [ ] 결제 웹훅 처리
- [ ] 예약 취소 + 환불 처리
- [ ] 예약 내역 조회 API
- [ ] 입실 안내 자동 문자 발송 (알리고 or 네이버 클라우드)

## 프론트엔드 (앱)

- [ ] 예약 입력 화면 — 날짜 선택 캘린더
- [ ] 예약 입력 화면 — 메시지 입력
- [ ] 결제 화면 — 금액 확인, 결제 수단 선택
- [ ] 예약 완료 화면
- [ ] Host 홈 — 예약 요청 목록
- [ ] Host — 예약 수락/거절 화면
- [ ] Guest 예약 내역 화면
- [ ] 예약 취소 화면

## 완료 기준

- Guest가 공간을 예약 요청할 수 있다
- Host가 수락/거절할 수 있다
- 결제가 완료된다
- 취소 및 환불이 처리된다

# Sprint 4 — 알림 + 리뷰 + 정산 (Week 7~8)

> 목표: 이용 완료 후 리뷰, Host 정산, 알림 시스템

## 백엔드

- [ ] Firebase FCM 푸시 알림 연동
- [ ] 알림 발송 (예약 요청, 확정, 취소, 리뷰 요청)
- [ ] 리뷰 작성 API
- [ ] 평균 평점 집계
- [ ] 정산 계좌 등록 API
- [ ] 월 정산 처리 (수동 → 자동화)
- [ ] 정산 내역 API

## 프론트엔드 (앱)

- [ ] 리뷰 작성 화면 (별점 + 텍스트)
- [ ] Host — 받은 리뷰 목록
- [ ] 알림 리스트 화면
- [ ] Host — 정산 계좌 등록 화면
- [ ] Host — 정산 내역 화면
- [ ] 마이페이지 (프로필, 설정)

## 완료 기준

- 이용 완료 후 리뷰를 쓸 수 있다
- 푸시 알림이 온다
- Host가 정산 계좌를 등록하고 수익을 확인할 수 있다

---

# Sprint 5 — QA + 버그 수정 (Week 9~10)

> 목표: 실제 사용 시나리오 전체 테스트

## 테스트 시나리오

- [ ] Host 회원가입 → 공간 등록 → 예약 수락 → 정산 수령 전체 플로우
- [ ] Guest 회원가입 → 검색 → 예약 → 결제 → 리뷰 작성 전체 플로우
- [ ] 예약 취소 시 환불 처리 확인
- [ ] 지도 검색 성능 테스트
- [ ] 결제 실패 케이스 처리 확인
- [ ] 푸시 알림 iOS / Android 동시 테스트

## 보안 체크

- [ ] 타인 예약 정보 접근 불가 확인
- [ ] 결제 데이터 암호화 확인
- [ ] 주소 비공개 (예약 확정 전) 확인
- [ ] API rate limiting 설정

---

# Sprint 6 — 출시 준비 (Week 11~12)

> 목표: 앱 스토어 출시 + 초기 Host 확보

## 출시 준비

- [ ] 앱 아이콘 / 스플래시 최종 디자인
- [ ] 앱스토어(iOS) 심사 제출
- [ ] 플레이스토어(Android) 제출
- [ ] 개인정보처리방침 페이지
- [ ] 이용약관 페이지
- [ ] 고객센터 이메일 / 카카오 채널 개설

## 초기 Host 확보 (동시 진행)

- [ ] 강남 스타트업 10개 직접 영업
- [ ] 마포·성수 IT 기업 10개 직접 영업
- [ ] 세무사·법무법인 사무실 10개 영업
- [ ] Host 온보딩 가이드 제작 (공간 사진 잘 찍는 법 등)

## 초기 마케팅

- [ ] 개발자·디자이너 커뮤니티 (OKKY, 디스콰이어드) 홍보
- [ ] 프리랜서 커뮤니티 포스팅
- [ ] 인스타그램 계정 개설 + 첫 포스팅

# MVP 완료 기준

> 아래 기준을 모두 만족하면 출시로 간주한다

| 기준 | 목표값 |
|---|---|
| 등록 공간 | 30개 이상 |
| 앱 스토어 출시 | iOS + Android 동시 |
| 핵심 플로우 | 오류 없이 동작 |
| 결제 | 실제 결제 테스트 완료 |
| 평균 응답 속도 | 3초 이내 |

# 기술 부채 (Post-MVP)

> MVP에서 의도적으로 제외한 것들 — 출시 후 순서대로 처리

1. 즉시 예약 (Host 수락 없이 자동 확정)
2. 시간 단위 예약
3. 회의실·교육장 카테고리
4. Host Premium 구독 상품
5. 광고 상품 (공간 상단 노출)
6. 법인 주소 서비스
7. All-Access 패스
8. 기업 복지 연계 B2B
9. 웹 버전 (PWA)
