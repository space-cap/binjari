# 1. 개요 (Overview)

## 제품 소개

**빈자리(Binjari)**는 기업·사무실의 유휴 업무공간(빈 책상)을 프리랜서·1인 기업·출장자와 연결하는 **양면 공간 중개 플랫폼**이다.

공유오피스를 직접 운영하지 않는다.
플랫폼을 통해 공간 등록 → 검색 → 예약 → 결제 → 이용 → 리뷰의 전 과정을 처리한다.

## 핵심 가치

> "남는 자리를 연결하다."

| 대상 | 가치 |
|---|---|
| **Host (공급자)** | 비어 있는 책상으로 추가 수익을 만든다 |
| **Guest (이용자)** | 카페보다 집중되고, 공유오피스보다 저렴한 진짜 사무실을 이용한다 |

# 2. 목표 (Goals)

## 비즈니스 목표

| 기간 | 목표 |
|---|---|
| MVP 출시 (3개월) | 서울 강남·마포·성수 Host 30개, Guest 300명 확보 |
| Year 1 | Host 100개, Guest 1,000명, 월 거래액 3,000만원 |
| Year 2 | Host 500개, Guest 5,000명, 월 거래액 1.5억원 |
| Year 3 | Host 5,000개, 전국 확장, 월 거래액 15억원 |

## 제품 목표

- 공간 등록부터 예약까지 **10분 이내** 완료
- 이용자 예약 완료율 **80% 이상**
- Host 온보딩 후 **첫 예약까지 7일 이내**
- 앱 스토어 평점 **4.5점 이상**

# 3. 사용자 (Users)

## 3-1. Host (공급자)

### Persona A — 중소기업 총무 담당자

| 항목 | 내용 |
|---|---|
| 역할 | 회사 총무팀, 사무관리 담당 |
| 상황 | 재택근무 확산으로 사무실 책상의 30~50%가 비어 있음 |
| 목표 | 유휴 공간을 수익화하여 임대료 부담을 줄이고 싶다 |
| 고통 | 어떻게 등록하는지 모르겠다, 법적 문제가 걱정된다 |
| 성공 기준 | 월 10만원 이상 추가 수익 발생 |

### Persona B — 전문직 사무실 대표

| 항목 | 내용 |
|---|---|
| 역할 | 세무사, 법무사, 노무사 등 전문직 사무소 대표 |
| 상황 | 오전/오후 특정 시간대 빈 책상이 항상 존재 |
| 목표 | 유휴 시간대 공간을 시간제로 수익화하고 싶다 |
| 고통 | 낯선 사람이 사무실에 오는 것이 불안하다 |
| 성공 기준 | 인증된 이용자만 입실, 월 20만원 추가 수익 |

### Persona C — 스타트업 대표

| 항목 | 내용 |
|---|---|
| 역할 | 팀 축소 또는 이전 후 남은 공간 보유 스타트업 |
| 상황 | 임대차 계약 기간이 남아 있어 임대료를 계속 내야 함 |
| 목표 | 남은 계약 기간 동안 빈 책상으로 임대료 회수 |
| 고통 | 복잡한 등록 절차 없이 빠르게 공간을 올리고 싶다 |
| 성공 기준 | 등록 후 1주일 내 첫 예약 |

---

## 3-2. Guest (이용자)

### Persona D — 독립 프리랜서

| 항목 | 내용 |
|---|---|
| 직군 | 개발자, 디자이너, 마케터 등 |
| 연령 | 25~38세 |
| 상황 | 재택 작업 시 집중이 어렵고, 카페는 전화가 안 된다 |
| 목표 | 주 3~5일 사용할 수 있는 조용한 사무 공간 |
| 고통 | 공유오피스는 너무 비싸고, 장기계약이 부담 |
| 예산 | 월 15~30만원 |
| 성공 기준 | 집 근처 10분 이내 실제 사무실 책상 이용 |

### Persona E — 1인 기업 창업자

| 항목 | 내용 |
|---|---|
| 상황 | 사업자등록은 되어 있지만 사무실이 없음 |
| 목표 | 클라이언트 미팅이 가능한 사무공간 + 법인 주소 |
| 고통 | 공유오피스 장기계약(6개월 이상)이 부담, 계약서가 복잡함 |
| 예산 | 월 20~50만원 |
| 성공 기준 | 필요할 때만 예약, 클라이언트에게 보여줄 수 있는 주소 |

### Persona F — 지방 출장 직장인

| 항목 | 내용 |
|---|---|
| 상황 | 본사는 부산, 서울 출장 시 하루~이틀 업무공간 필요 |
| 목표 | 당일 즉시 예약 가능한 서울 내 업무공간 |
| 고통 | 카페·호텔 로비는 집중이 안 됨, 공유오피스는 일 단위 예약이 어려움 |
| 예산 | 일 1~3만원 |
| 성공 기준 | 앱에서 검색 → 10분 안에 예약 완료 |

# 4. 사용자 여정 (User Journey)

## Host 여정

```
[공간 발견] 지인 추천 or 검색 or 영업팀 컨택
     ↓
[회원가입] 사업자 인증 필수
     ↓
[공간 등록] 사진, 위치, 가격, 이용 규칙 설정
     ↓
[검토 완료] 빈자리 운영팀 승인 (24시간 내)
     ↓
[예약 수신] 앱 알림 → 수락/거절
     ↓
[이용자 입실] 자동 안내 문자 발송
     ↓
[정산] 월 1회 자동 정산
     ↓
[리뷰 수신] 이용자 후기 확인
```

## Guest 여정

```
[공간 발견] 검색 or SNS 광고 or 지인 추천
     ↓
[회원가입] 소셜 로그인 (카카오/구글)
     ↓
[공간 검색] 지도 기반 → 필터 (가격, 거리, 업종)
     ↓
[공간 상세] 사진, 리뷰, 시설 정보 확인
     ↓
[예약 요청] 날짜/기간 선택 → 결제
     ↓
[예약 확정] Host 수락 → 확정 알림
     ↓
[입실] QR 코드 or 안내 문자로 입실
     ↓
[이용] 업무 진행
     ↓
[리뷰 작성] 별점 + 후기 작성
```

# 7. MVP 범위 (MVP Scope)

> MVP 목표: 강남·마포·성수 지역 Host 30개, Guest 300명 확보
> MVP 개발 기간: 3개월

## In Scope ✅

| 기능 | 설명 |
|---|---|
| 회원가입·로그인 | 카카오 소셜 로그인, 사업자 인증 |
| 공간 등록 | 사진, 위치, 가격, 규칙, 가용 일정 |
| 공간 검색 | 지도 기반, 날짜 필터, 가격 필터 |
| 공간 상세 | 사진, 정보, 리뷰, 위치 |
| 예약 요청 | 날짜 선택, 메시지, 결제 |
| 결제 | 카드, 카카오페이 |
| 예약 관리 | Host 수락/거절, Guest 예약 내역 |
| 알림 | 예약 요청·확정·취소 푸시 알림 |
| 리뷰 | 별점 + 텍스트 작성 |
| 정산 | 계좌 등록, 월 정산 |

| 파일 저장 | **개발:** 로컬 스토리지 / **운영:** Oracle Cloud Object Storage | 개발 단순화, 운영 비용 효율 |
| 푸시 알림 | Firebase FCM | iOS/Android 통합 |
| 인프라 | **개발:** 로컬 / **운영:** Oracle Cloud Infrastructure (OCI) | 무료 티어 활용 가능 |
| 사업자 인증 | 국세청 API | 사업자등록번호 진위 확인 |

## 환경별 파일 저장 전략

| 환경 | 저장소 | 설정 |
|---|---|---|
| **개발 (local)** | 로컬 디스크 (`/uploads`) | 별도 클라우드 계정 불필요, 빠른 개발 |
| **운영 (production)** | Oracle Cloud Object Storage | OCI 무료 티어 20GB 제공, S3 호환 API 지원 |

> **팁:** OCI Object Storage는 AWS S3 호환 API를 지원하므로, 환경 변수(`STORAGE_TYPE=local|oci`)로 전환만 하면 코드 변경 없이 동작한다.

# 10. 수익 모델 (Revenue Model)

## 기본 방침: 무료 중개

> 당근마켓처럼 Host-Guest 간 거래에 **수수료를 받지 않는다.**
> 초기 네트워크 확보를 최우선으로 하고, 트래픽이 쌓인 후 광고·프리미엄으로 수익화한다.

### 왜 무료인가?

- Host 진입 장벽 제거 → "수수료 없다" 한 마디로 온보딩 가속
- Guest 이용 비용 절감 → 공유오피스·카공족 대비 가격 경쟁력 극대화
- 빠른 네트워크 효과 → 공급·수요 동시 성장
- 입소문 마케팅 → 수수료 없는 플랫폼은 자연스럽게 바이럴

---

## 수익 구조

### Phase 1 — 광고 수익 (트래픽 확보 후)

| 상품명 | 설명 | 예상 가격 |
|---|---|---|
| **공간 상단 노출** | 지역 검색 시 상단 3자리 고정 노출 | 월 3~10만원 |
| **추천 공간 배너** | 홈·검색 결과 내 추천 슬롯 | 월 5~15만원 |
| **지역 타겟 광고** | 특정 지역 이용자에게만 노출 | 월 5~20만원 |

### Phase 2 — Host Premium 구독

| 플랜 | 가격 | 혜택 |
|---|---|---|
| **Free** | 무료 | 공간 1개 등록, 기본 예약 관리 |
| **Pro** | 월 19,000원 | 공간 5개, 통계 대시보드, 우선 노출 |
| **Business** | 월 49,000원 | 무제한 공간, API 연동, 전담 매니저 |

### Phase 3 — 기업 서비스 (B2B)

| 상품 | 설명 |
|---|---|
| 임직원 복지 패키지 | 기업 복지 예산으로 빈자리 이용권 구매 |
| 기업 전용 공간 계약 | 팀 단위 장기 이용 계약 |

## 수수료 도입 시점 (선택적)

> 무료 중개를 원칙으로 하되, 아래 조건 충족 시 검토한다.

| 조건 | 내용 |
|---|---|
| 도입 시점 | MAU 10,000명 이상 + 등록 공간 1,000개 이상 달성 후 |

# 12. 성공 지표 (Success Metrics)

> 무료 중개 모델이므로 거래액보다 **사용자 수·참여도**가 핵심 지표다.

## 핵심 KPI

| 지표 | 정의 | MVP 목표 | Year 1 목표 |
|---|---|---|---|
| 등록 공간 수 | 플랫폼 내 활성 공간 수 | 30개 | 100개 |
| MAU (Guest) | 월간 활성 이용자 수 | 300명 | 5,000명 |
| 재방문율 | 월 2회 이상 방문 Guest 비율 | 30% | 50% |
| 예약 완료율 | 예약 요청 → 확정 비율 | 70% | 80% |
| Host 활성율 | 월 1건 이상 예약 받은 Host 비율 | 50% | 70% |
| 앱 평점 | 앱스토어/플레이스토어 | 4.0+ | 4.5+ |
| NPS | 순추천지수 | 30+ | 50+ |
| 광고 수익 (Phase 1) | 월 광고 매출 | 0원 | 500만원 |
| Pro 구독자 수 (Phase 2) | Host Premium 유료 구독 | 0명 | 50명 |

## 보조 지표

- 공간 상세 → 예약 전환율 (목표: 15% 이상)
- Host 온보딩 → 첫 예약까지 걸리는 시간 (목표: 7일 이내)
- CS 문의 건수 / 예약 건수 비율 (목표: 5% 이하)
- 앱 DAU/MAU 비율 (목표: 20% 이상 = 스티키니스)

# 13. 리스크 및 가정

## 주요 리스크

| 리스크 | 영향 | 대응 방안 |
|---|---|---|
| Host 확보 부족 | 서비스 가치 저하 | 초기 B2B 영업팀 집중 투입 |
| 전대차 법적 분쟁 | 서비스 중단 | 사업자 인증 + 전대차 동의 확인 필수화 |
| 공간 품질 편차 | 이용자 신뢰 하락 | 등록 기준 강화, 운영팀 승인 프로세스 |
| 결제 사고 | 신뢰도 타격 | PCI DSS 준수 PG사 사용 |
| 경쟁사 진입 | 시장 점유율 감소 | Host 락인 전략, 빠른 공급 확보 |

## 가정

- 사업자등록이 완료된 공간만 등록 허용
- 전대차 동의를 받은 공간만 등록 가능 (Host 서약)
- MVP는 서울 3개 지역(강남·마포·성수)에 집중
- 초기 6개월은 B2B 영업팀이 Host를 직접 확보
